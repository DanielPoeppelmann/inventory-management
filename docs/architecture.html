Data Flow — Request Lifecycle

1

User interacts with a View

A view such as Inventory.vue mounts and the user adjusts the shared FilterBar (warehouse, category, status, time period).

2

Filter state is resolved

The useFilters composable holds global filter state and returns the active set via getCurrentFilters().

3

API client issues the request

api.js builds the query string and sends an axios request, e.g. GET /api/inventory?warehouse=Tokyo&category=Sensors to http://localhost:8001.

4

FastAPI filters in memory

The matching endpoint in main.py applies apply_filters() / filter_by_month() over the in-memory lists loaded from mock_data.py.

5

Pydantic validates the response

Results are serialized through Pydantic response models, guaranteeing a strict schema before returning JSON.

6

Vue reactivity re-renders

axios resolves the promise, the view assigns the data to reactive refs, and derived computed properties update the DOM — no extra calls needed for detail modals.
